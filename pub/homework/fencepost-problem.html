The Fencepost Error

Problems marked with an asterisk (*) are considered to be challenge problems. You should still try them, but we will discuss them in class.

Problem 1: Building a Fence

I want to build a 100 m100\,\mathrm{m}100m fence. The fenceposts should be 2 m2\,\mathrm{m}2m apart and there should be a post at each end. How many fenceposts are needed?

Solution

We know that there need to be 1002\frac{100}{2}2100​ two-meter sections in order to span the required length. But there is going to be one more post than that, because we need a post on both ends. Therefore we require 515151 fenceposts.

Problem 2: Marbles Arranged in a Line

100100100 marbles are arranged in a line. The first marble is red, the second is blue, and the third is yellow. This pattern repeats: the marbles are red, then blue, then yellow, then red again, then blue again, then yellow again, etc. How many red marbles are there?

Solution

We know there are going to be 333333, the floored quotient of 100100100 by 333, full sections of red, blue, and yellow marbles. But this is only enough to account for 999999 of the 100100100 marbles. There should be an additional red marble at the end, making a total of 343434 red marbles.

Problem 3: Linear Range

I wrote down a increasing sequence of rational numbers starting with 101010 and ending with 100100100. Every two terms in this sequence are 14\frac{1}{4}41​ apart. That is, this sequence starts 10,10.25,10.5,10.75,11,…10, 10.25, 10.5, 10.75, 11, \dots10,10.25,10.5,10.75,11,…. How many numbers did I write?

Solution

This is basically a fencepost problem, where the sections are of length 14\frac{1}{4}41​ and each number I write represents a fencepost. We expect (100−10)÷14=360(100 - 10) \div \frac{1}{4} = 360(100−10)÷41​=360 sections, and so there should be 361361361 numbers written.

Problem 4: Implicit Binary Tree*

The array [1,2,3,…,15][1, 2, 3, \dots, 15][1,2,3,…,15] can be drawn as a compact binary tree which satisfies the following properties:

  • Each node of the tree has a unique label from the array.

  • From every node there are either two branches going up the tree (called the left and right branches), or no outgoing branches.

  • The tree has a height of 333. That is, all of the highest nodes of the tree are only 333 edges away from the root.

  • If a node has outgoing branches, then its label is greater than all labels in the left subtree, and less than all labels in the right subtree.

Now we will draw a binary tree satisfying the three requirements, but with the bigger array [1,2,…,127][1, 2, \dots, 127][1,2,…,127] and a height of 666 instead of 333. Starting with the root, go left, then right, then left, then right. What is the label for this node?

Solution

Note that 127+1=27=1+2+4+8+16+32+64127+1=2^7=1+2+4+8+16+32+64127+1=27=1+2+4+8+16+32+64. This means that the tree meeting the requirements is going to be full at every layer, up to height 666. There have to be 127−12=63\frac{127-1}{2} = 632127−1​=63 total nodes on either side of the root node, so that together with the root node we will get 127127127 nodes in total. This means that the left subtree of the root node is labeled with numbers from 111 to 636363, the root is labeled 646464, and the right subtree is labeled with numbers from 656565 to 127127127.

Repeating this argument, we find that the 2nd node we visit (directly left of the root) must be labelled 323232. The 3rd node we visit will be labelled 484848. The 4th node we visit will be labelled 404040. The 5th, and final, node that we visit will be labelled 444444.
